# Protocol: 75 自驱卡片与强关系商业网络

版本日期：2026-05-04

## 1. 本文定位

本文记录 Protocol: 75 在 2026-05-04 讨论形成的新方向，重点回答三个问题：

1. 在不做质押、押金、保证金、现金奖励的情况下，用户为什么会自驱参与；
2. 在早期没有商户权益、没有大规模品牌合作的情况下，平台如何名正言顺地收取极低费用；
3. 行动局、组卡和强关系圈层如何形成区别于传统流量平台的商业网络。

核心结论：

> Protocol: 75 的早期发动机不是金钱奖励，而是“履约卡 + 总履约力 + 小组高阶卡 + 社交展示 + 认证入册”形成的自驱系统。

> Protocol: 75 的商业潜力不是卖陌生流量，而是让商户把权益交给已经被共同履约验证过的强关系圈层。

## 2. 不做质押后的动力问题

原 Web3/Aptos 版本中，用户通过质押形成强约束，失败会扣分或损失资金。

中国大陆合规版本中，Protocol: 75 明确不做：

- 质押；
- 抵押；
- 保证金；
- 对赌；
- 罚金；
- 资金池；
- 底层生息；
- 用户之间输赢分配。

这会带来一个问题：

> 如果用户只付 0.99 元、9.9 元或 19.9 元，成本很低，放弃成本也很低，怎样让用户真的在乎？

新版答案是：

> 把金钱损失转化为身份损失、收藏损失、连续性损失和小组高阶奖励损失。

用户害怕失去的不是钱，而是：

- 本周限时卡；
- 本月套装进度；
- 创始季卡片；
- 连续履约记录；
- 总履约力增长；
- 高阶卡风；
- 小组钻石卡；
- 个人主页展示完整度；
- 被同组成员看到自己掉队。

这套机制的心理基础是：稀缺、连续、收藏、展示、归属和 FOMO。

## 3. 早期收费原则：行动免费，认证付费

早期不建议在开局时收费。

原因：

- 开局收费会增加传播摩擦；
- 用户会觉得每次行动都被平台抽税；
- 民间扩散和模板复制需要低门槛；
- 早期品牌势能不足，不适合强行收高价。

推荐模型：

> 行动免费，认证付费。

免费阶段包括：

- 创建行动局；
- 加入行动局；
- 每日打卡；
- 小组见证；
- 行动局内结果查看；
- 基础记录留存。

付费阶段发生在完成之后，用户如果希望把本次结果正式沉淀为可展示、可验证、可授权的身份资产，则支付极低的认证入册费。

付费后获得：

- 正式履约卡；
- 平台签名；
- 二维码验证页；
- 进入个人公开卡册；
- 增加总履约力；
- 参与套装和赛季图鉴；
- 可授权展示给外部；
- 可用于后续商户权益或组织验证。

一句话：

> 用户不是为打卡付费，而是为“我完成了，我要让它正式留下来”付费。

## 4. 认证入册费设计

早期收费必须极低、透明、提前告知。

建议价格：

| 项目 | 建议价格 | 说明 |
| --- | ---: | --- |
| 基础认证入册 | 0.99 元/张 | 生成正式履约卡、签名、入册、加总履约力 |
| 多源证据认证 | 2.99 元/张 | 聚合 Keep、薄荷、健康 App、小组见证等多个证据源 |
| 月度套装结算 | 6.9 元/月 | 对当月卡片进行套装结算、生成月度徽章 |
| 月度认证会员 | 9.9 元/月 | 当月包含若干次认证入册 |
| 创始季通行证 | 75 元/年 | 早期限量，包含认证额度和创始身份，不卖战力 |

必须遵守：

- 不付费也能看到基础结果；
- 付费只解锁正式签名、公开展示、入册和可验证凭证；
- 不做心理绑架式高价付费；
- 不在用户投入大量时间后突然索要高价；
- 价格提前展示；
- 卡片稀有度不由付费决定。

## 5. 为什么这个收费名正言顺

平台收的不是“成功费”，也不是“奖励费”，而是：

> 规则结算、凭证签名、二维码验证、长期入册、公开展示和身份档案维护的技术服务费。

用户实际购买的是：

- 正式认证；
- 平台签名；
- 长期保存；
- 公开验证；
- 可授权展示；
- 身份系统入账。

这比局长工具包、高级规则包、高级卡面更适合作为早期收入。

原因：

- 局长早期应该免费，承担传播角色；
- 高级卡面要等品牌势能起来后才有价值；
- 高级规则包需要专业资源背书；
- 商户、API、组织后台都是后置商业增强层；
- 只有认证入册和用户完成后的成就感是早期最自然的付费时刻。

## 6. 履约卡的本质：奖牌，不是艺术品

不可交易的卡片要有价值，不能靠“艺术品价值”，而要靠“履约证明价值”。

一个不知名艺术生随便画一张卡，用户不会在乎。

但一个人真的和 3-7 个朋友完成了 21 天早睡、30 天慢跑、75 天身心自律，这张卡就不是普通图片，而是：

> 完成某件困难事情后的可验证奖牌。

履约卡的价值来自：

- 它证明了真实行为；
- 它只能完成行动局获得；
- 它有时间稀缺性；
- 它有难度稀缺性；
- 它有证据等级；
- 它有小组上下文；
- 它有编号、赛季和卡册位置；
- 它能展示和比较。

Protocol: 75 要让卡片像奖牌、赛季皮肤、游戏战力、GitHub 贡献图、马拉松完赛证书一样被用户在乎。

## 7. 卡片品质与卡风

Protocol: 75 可以参考集卡游戏的品质感，但不能参考抽卡付费逻辑。

核心原则：

> 卡片稀有度只能由履约难度、证据等级、时间窗口、完成质量和小组协作决定，不能由充值决定。

### 7.1 卡片品质

卡片品质代表履约难度和完成质量。

可用命名：

| 游戏化命名 | 更克制命名 | 获取条件示例 |
| --- | --- | --- |
| 普通 | 基础 | 7 天局，基础完成，E0/E1 证据 |
| 稀有 | 进阶 | 14/21 天局，完成率达标 |
| 史诗 | 精英 | 30 天局，高完成率，E2+ |
| 传说 | 传奇 | 75 天局，E3+，小组表现优秀 |
| 神话/创始 | 创始/限定 | 创始季、特殊赛季、极高难度或长期连续 |

### 7.2 卡风

同一张卡功效一样，但外观不同。

可用卡风：

- 普通版：完成基础要求；
- 金边版：完成率 90%+；
- 星辉版：全程无补签；
- 异画版：创始季、特殊赛季、节气主题；
- 钻石版：高证据等级 + 小组全员完成；
- 纪念版：年度、季度、城市、校园、社群主题。

注意：

- 卡风来自行为成就，不来自充值；
- 不做随机抽卡；
- 不做付费开包；
- 不做概率稀有；
- 所有卡都必须来自明确履约。

## 8. 限时卡与 FOMO

在没有交易属性和金融属性的前提下，卡片 FOMO 来自时间稀缺和社交展示。

推荐设计：

- 周卡：例如 `2026-W19 睡眠稳定卡`，过期永不复刻；
- 月卡：集齐当月若干周卡后解锁；
- 季度卡：连续三个月完成指定套装后解锁；
- 创始季卡：早期用户专属，后续不复刻；
- 节气卡：围绕立春、夏至、秋分等主题；
- 城市/校园卡：特定地点或社群主题；
- 小组纪念卡：同一个行动局全员完成后发放。

关键机制：

- 错过的卡在图鉴中显示灰色空位；
- 空位不能补正卡；
- 可以通过修复局获得“修复卡”；
- 正卡保持稀缺；
- 修复卡给用户台阶。

用户害怕的不是损失钱，而是：

> 我的卡册永远空了一格。

## 9. 组卡：强关系约束的核心

小组机制是 Protocol: 75 的关键。

除了裂变，小组还承担三个功能：

1. 强关系约束；
2. 共同履约见证；
3. 未来商业圈层传播。

高阶卡片应由个人履约和小组协作共同决定。

推荐机制：

| 情况 | 结果 |
| --- | --- |
| 个人完成，组内有人失败 | 个人普通卡 |
| 个人完成率 90%+ | 个人金边卡 |
| 小组 80% 成员完成 | 小组协作版 |
| 小组全员完成 | 钻石小组版 |
| 小组全员完成 + 无补签 | 星辉钻石版 |
| 小组全员完成 + 高证据等级 | 高证据钻石版 |

这样设计的好处：

- 一个人失败不会让其他人颗粒无收；
- 但小组全员完成才能解锁最高阶卡风；
- 每个人都会希望队友坚持；
- 小组成员会互相提醒、鼓励和监督；
- 强关系约束替代了金钱惩罚。

一句话：

> 不是我失败了会被罚，而是我失败了会让全组错过高阶卡。

这就是原 Web3 版本强社会关系约束的合规转译。

## 10. 小组纪念卡

当一个行动局全员完成时，每个成员除了个人履约卡，还可以获得同一编号的小组纪念卡。

小组纪念卡包含：

- 行动局编号；
- 小组成员昵称；
- 组长标识；
- 行动局日期；
- 全员完成率；
- 规则包版本；
- 小组证据等级；
- Squad Perfect Clear 或类似标识；
- 同一张组卡的关联编号。

它的价值是：

> 我们一起完成过一件事。

这类似一起跑完马拉松的奖牌，也类似一起打赢一场游戏副本的成就。

## 11. 组长激励

组长不应该被收费。

组长承担组织成本，是早期传播节点。

组长可以获得：

- 组长专属卡面；
- 组长徽章；
- 组长经验值；
- 优先开高阶局资格；
- 小组全员完成时的额外卡风；
- 商户权益中的体验升级；
- 创始局长身份。

不建议：

- 现金返佣；
- 层级分销；
- 拉人返现；
- 收益承诺。

组长不是大 V，不靠粉丝崇拜，而是小范围组织者。

一句话：

> 大 V 分发依赖影响力，组长分发依赖信任关系。

## 12. 展示系统

如果没有展示，卡片系统就不会成立。

必须有：

- 个人主页；
- 总履约力；
- 领域雷达图；
- 近期活跃曲线；
- 卡册；
- 空缺图鉴；
- 赛季图鉴；
- 小组纪念页；
- 最高品质卡展示；
- 最近获得卡片；
- 分享海报；
- 二维码验证页。

用户主页可以展示：

- 总履约力；
- 本赛季卡册完成度；
- 最高品质卡；
- 连续履约周数；
- 领域雷达图；
- 最近获得的 3 张卡；
- 参与过的行动局；
- 组卡记录。

用户会在乎的核心心理是：

> 提升战力的目的，是让别人看见和羡慕。

但这个战力必须来自真实履约，而不是氪金。

## 13. 早期用户策略

早期最容易启动的人，不是完全没有自律动力的人，而是已经有记录欲、展示欲、自律人设的人。

他们可能已经在：

- Keep；
- 薄荷；
- Apple Health；
- 微信运动；
- 小红书；
- 朋友圈；
- 跑团；
- 自习室；
- 健身社群；
- 早睡/戒熬夜群。

他们本来就在展示：

- 我今天跑了；
- 我又早起了；
- 我连续打卡 30 天；
- 我今年状态很好。

Protocol: 75 给他们一个更强的表达：

> 你的自律不再是一堆截图，而是一套持续增长的战力、卡册和履约身份档案。

第一阶段先抓这批人。

然后，通过他们展示出来的卡片、组卡、战力和行动局纪念页，影响那些坚持困难但有向往的人。

路径：

1. 自律展示型用户先来；
2. 他们完成行动局并生成卡片；
3. 他们分享主页和组卡；
4. 旁观者产生羡慕和 FOMO；
5. 旁观者复制行动局；
6. 小组带动原本坚持困难的人进入。

## 14. 与传统流量平台的对比

Protocol: 75 的商业叙事不能只说“降本增效”，还要讲出与传统流量平台的结构性差异。

| 传统流量平台 | Protocol: 75 |
| --- | --- |
| 商户买曝光 | 商户发布权益 |
| 用户是陌生流量 | 用户是履约圈层 |
| 依赖大 V/KOL | 依赖组卡和强关系小组 |
| 预算进入营销漏斗 | 权益进入真实行动圈层 |
| 粉丝关系弱，容易情绪化 | 小组关系来自共同履约 |
| 算法黑箱分发 | 用户主动授权分发 |
| 海投后赌转化 | 从已验证人群开始扩散 |
| 平台掌握关系 | 用户掌握关系和凭证 |

关键表达：

> 传统平台让商户把钱投给流量中心，Protocol: 75 让商户把权益交给真实履约圈层。

> Protocol: 75 不是流量平台，而是可信履约圈层网络。

## 15. 打麻将式的强关系传播

通俗地说，Protocol: 75 的关系网络有点像打麻将。

你找到一个喜欢打麻将的人，往往就能找到一群喜欢打麻将的人。

同理：

- 找到一个喜欢跑步的人，往往能找到一个跑步圈；
- 找到一个喜欢打羽毛球的人，往往能找到一群球友；
- 找到一个喜欢台球的人，往往能找到一批台球搭子；
- 找到一个认真自律的人，往往能找到一批正在尝试自律的人。

而且这种关系是可流动、可融入的。

一个人今天可以和 A 打羽毛球，明天也可以和 B 打羽毛球。即便一开始和 B 是陌生人，打完以后也可能变成熟人。

行动局也是如此：

- 一次行动局让陌生人变成同伴；
- 一张组卡记录他们共同完成过的事；
- 下一次行动局又可以连接新的成员；
- 关系像链路一样逐层扩散。

这不是靠大 V 统一管理，也不是靠平台中心化分发，而是民间小组自然扩散。

## 16. 可信关系图谱

一个已经共同履约成功的小组，不只是几个人一起打过卡，而是被验证过：

- 他们曾经共同参与一个行动局；
- 他们在同一规则下坚持过一段时间；
- 他们互相见证过；
- 他们一起完成过结算；
- 他们拥有同一张组卡或同一组编号的组卡。

这说明他们之间有强关联，不是陌生流量。

因此：

> 组卡 = 可验证的强关系圈层凭证。

这张组卡在未来商业活动中可以成为权益进入圈层的通行证。

## 17. 用户主动，商户被动

Protocol: 75 必须避免商户骚扰用户。

平台不应该让商户沿着关系图谱主动私信用户，也不应该向商户出售用户名单。

正确设计是：

> 商户发布权益，用户主动领取，用户主动带入圈层。

商户可以发布：

- 本地权益；
- 组团体验；
- 持卡福利；
- 行动局赞助；
- 线下活动名额。

但商户默认只能进入公共权益池，不能主动直达用户。

用户可以选择：

- 自己领取；
- 分享给某个行动局；
- 发起一个带权益的新行动局；
- 邀请历史队友参与；
- 完全忽略。

关键原则：

- 用户拥有凭证；
- 用户拥有关系；
- 用户拥有授权权；
- 商户只能被动发布机会；
- 用户决定机会是否进入自己的圈层。

## 18. 圈层权益传播

当商户发布权益后，组长或高履约用户可以主动把权益带入自己的行动圈层。

示例：

- 轻食店发布“运动组卡福利”；
- 某个 21 天慢跑局组长领取；
- 他分享给本组成员；
- 如果 5 人一起到店，组长获得额外饮品或升级权益；
- 组员获得统一折扣；
- 商户获得一组真实运动人群。

这不是平台强推广告，而是：

> 用户把权益带入自己已经验证过的强关系圈层。

组长获得一点额外好处是合理的，因为他承担组织成本。

但奖励应以非现金权益为主：

- 到店升级；
- 免费饮品；
- 体验名额；
- 会员天数；
- 卡面徽章；
- 局长经验值。

不做现金返佣，不做层级分佣，不做裂变分销。

## 19. 商业闭环

未来商业闭环可以是：

1. 小组完成行动局；
2. 获得组卡；
3. 组卡证明强关系；
4. 商户发布权益；
5. 组长或成员主动领取；
6. 权益进入圈层；
7. 圈层组团消费或参与；
8. 商户获得高质量用户；
9. 用户获得权益；
10. 平台收取认证、核销或技术服务费。

核心表达：

> 组卡不是奖励，而是未来权益进入强关系圈层的通行证。

## 20. 防骚扰与合规边界

平台必须保护用户主权：

- 不向商户出售用户名单；
- 不允许商户私信未领取用户；
- 不允许商户沿关系图谱主动骚扰；
- 用户领取权益后才建立临时联系；
- 用户可以随时撤回授权；
- 商户发布权益有频率限制；
- 用户可以关闭某类权益推荐；
- 平台默认只展示匿名画像和凭证条件。

用户授权前，商户只能看到：

- 匿名数量；
- 行为类型；
- 凭证等级分布；
- 大致区域；
- 是否有人授权接收相关权益。

用户授权后，也只展示必要信息。

一句话：

> 商户不是捕猎用户，而是摆出一个足够好的权益，让用户自己选择是否靠近。
