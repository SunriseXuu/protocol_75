# Protocol: 75：AI 驱动的可信持续行为凭证协议

面向中国大陆合规落地与政府创业补贴答辩版本

版本日期：2026-04-29

## 0. 执行摘要

Protocol: 75 不是再做一个普通打卡 App，而是面向长期目标履约、习惯养成、校园活动、训练营社群、本地小微商户和品牌权益投放场景，建设一套轻量、合规、可验证的可信持续行为凭证协议。

项目通过 AI 计划生成、行为数据校验、异常检测、社区见证、多方签名、哈希存证和用户授权展示等技术，把用户在运动、学习、睡眠、压力管理等长期目标中的真实坚持，转化为可验证、可撤销、可授权展示的“持续行为凭证”。品牌、校园、机构、社区和本地商户可以基于这些凭证识别高质量用户，直接发放权益、服务名额和优惠资源，减少传统投流中无效曝光、虚假流量和转化漏斗损耗。

项目坚持四条合规底线：

- 不做个人征信，不为金融授信、借贷、保险定价提供信用评分。
- 不做金融业务，不涉及代币、稳定币、质押、对赌、收益承诺或资金池。
- 不代收代付用户奖励资金，早期以优惠券、服务权益、商户直付和持牌机构托管为主。
- 不将区块链作为唯一底层，只将其作为高等级凭证的外部存证选项，且在正式对公众提供相关服务前按监管要求完成备案与安全治理。

## 1. 一句话概括

Protocol: 75 是一套 AI 驱动的可信持续行为凭证协议，帮助用户把长期坚持转化为可验证的行为证明，帮助商家和机构基于真实行为凭证直接触达高质量用户，尤其帮助小微商户在本地生活半径内降低获客成本并提升权益投放效率。

## 2. 项目描述

### 2.1 这是一个什么东西

项目面向“长期坚持很难证明、优质用户很难识别、商户投放成本高”的问题，提供一套可插拔的可信行为凭证系统。

用户可以在不同场景中参与履约任务，例如：

- 30 天晨跑或运动挑战；
- 21 天学习打卡或职业技能训练；
- 14 天睡眠改善计划；
- 7 天压力管理、正念练习或情绪记录；
- 校园社团、训练营、健身房、课程机构组织的主题挑战。

平台并不直接宣称用户“信用好”或“人品好”，而是只证明一组有限事实：

> 在某个规则版本、某个时间范围、某类数据来源、某个证据等级下，该用户完成过某项可验证行为。

这种表达比传统“大而全的信用分”更小、更稳、更适合三人创业团队早期落地，也更符合数据合规与监管边界。

### 2.2 用户怎么用

用户端：

- 选择长期目标，例如运动、学习、睡眠、压力管理；
- 授权必要数据，例如打卡记录、设备数据、社区活动记录；
- 获得 AI 生成的阶段计划、提醒和复盘；
- 完成目标后获得“持续行为凭证”；
- 自主决定是否向品牌、校园、社群或机构展示凭证，换取优惠券、课程名额、活动资格、会员权益或其他非金融权益。

社区或合作方端：

- 通过后台或 API 创建挑战活动；
- 对用户完成情况进行见证签名；
- 将本社区已有用户体验和运营能力接入平台；
- 获得标准化凭证生成、风控校验和数据看板能力。

B 端商户端：

- 选择目标人群，例如连续运动 30 天、完成某课程训练营、长期睡眠改善用户；
- 设置权益预算，例如优惠券、服务包、课程体验名额、品牌活动资格；
- 通过验证接口或二维码确认用户凭证真实性；
- 直接向用户发放权益，避免传统广告投放中的虚假流量与低转化浪费。
- 本地小微商户可以只服务自己熟悉的地理半径，例如学校周边、社区周边、健身房周边、商圈周边，不需要承担大平台高额投流成本。

平台端：

- 提供 AI 计划、AI 风控、凭证签发、凭证验证、撤销管理、合作方评级、数据看板和 API 服务；
- 不保管用户奖励资金；
- 不向第三方出售个人原始数据；
- 不生成用于金融活动的个人信用评价。

### 2.3 核心产品形态

早期产品可由四个模块组成：

| 模块 | 说明 | 早期优先级 |
| --- | --- | --- |
| 用户凭证钱包小程序 | 用户查看、管理、授权展示自己的行为凭证 | 高 |
| 合作方活动后台 | 社区、社团、训练营、健身房创建挑战并提交见证数据 | 高 |
| 凭证验证页/API | 商户扫码或系统调用接口验证凭证真假、等级、有效期 | 高 |
| AI 风控与运营看板 | 异常识别、完成率分析、人群分层、权益投放效果分析 | 中高 |

这一路线允许项目早期不从零自建完整 C 端社区，而是先接入已有社群、课程、校园活动、健身房和训练营，用 API 和后台帮助他们发行标准化可信凭证。

### 2.4 可信行为凭证体系

项目采用“证据等级 E + 存证等级 A”的二维体系，避免把“行为本身是否真实”和“记录之后是否被篡改”混为一谈。

#### 证据等级 E：证明行为本身有多可信

| 等级 | 名称 | 数据来源 | 适用场景 |
| --- | --- | --- | --- |
| E0 | 用户自证 | 用户自行打卡、文字、图片、手动记录 | 个人记录、低价值徽章 |
| E1 | 社区见证 | 社群、社团、训练营、线下机构对完成情况签名 | 校园活动、低门槛权益 |
| E2 | 平台风控 | 平台规则校验、AI 异常检测、重复行为识别、图片/文本审核 | 一般品牌权益、活动资格 |
| E3 | 可信数据源 | 穿戴设备、手机健康数据、学习平台、企业系统、机构系统等 | 高价值优惠、会员权益 |
| E4 | 多方交叉证明 | 用户授权 + 社区见证 + 可信数据源 + 平台风控共同成立 | 高价值合作、重点活动、争议可审计场景 |

#### 存证等级 A：证明记录之后有没有被篡改

| 等级 | 名称 | 技术方式 | 说明 |
| --- | --- | --- | --- |
| A0 | 平台记录 | 传统数据库记录 | 成本最低，适合 MVP |
| A1 | 哈希日志 | 每条记录生成哈希，形成追加式日志 | 可证明记录被修改会留下痕迹 |
| A2 | Merkle Root + 多方签名 | 每日或每批记录生成 Merkle Root，由平台与合作方共同签名 | 适合合作方共同见证 |
| A3 | 外部存证 | 合规联盟链、第三方时间戳、司法/公证类存证或其他可信存证服务 | 用于高价值凭证和争议处理 |

重要原则：

- 上链或外部存证不能把低质量数据变成高质量数据。
- 一个用户自证的 E0 凭证，即使达到 A3 存证，也只是“低证据等级、高存证强度”。
- 高价值商业合作优先使用 E3/A2 或 E4/A3 凭证。
- 商户可以根据自己的权益价值，自主选择接受哪一等级的凭证。

### 2.5 凭证内容示例

一张凭证不是简单的“分数”，而是结构化事实：

```json
{
  "credential_type": "30_day_running_consistency",
  "holder": "user_did_or_internal_id",
  "issuer": "Protocol75 Growth Passport",
  "witness": "campus_running_club_a",
  "rule_version": "v1.2",
  "period": "2026-05-01 to 2026-05-30",
  "evidence_level": "E3",
  "anchoring_level": "A2",
  "status": "valid",
  "expires_at": "2026-11-30"
}
```

商户看到的不是用户全部隐私，而是最小化后的验证结果：

- 凭证是否真实；
- 凭证是否过期；
- 凭证是否被撤销；
- 凭证对应的行为类型、规则版本和等级；
- 是否满足商户设置的权益领取条件。

### 2.6 核心技术亮点

第一，可信行为凭证引擎。  
平台将行为数据、合作方见证、设备数据、AI 风控结果和存证证明组合为标准化凭证，形成可复用的底层能力。

第二，AI 计划与陪跑系统。  
用户输入目标后，AI 根据目标难度、时间、历史完成率生成阶段计划，提供提醒、复盘、调整建议和激励反馈，提升坚持率。

第三，AI 风控与反作弊系统。  
平台通过规则引擎、异常检测、多模态审核和设备数据校验识别异常行为，例如短时间异常步数、重复上传图片、批量打卡、异常时间分布、社区批量作假等。

第四，多方签名与可审计日志。  
用户、社区、平台可以分别对授权、见证、验证结果进行签名。平台采用哈希日志、Merkle Tree 和外部存证，证明凭证生成过程没有被事后篡改。

第五，用户授权与隐私最小化。  
用户拥有凭证展示权。商户只获得验证结果，不默认获得用户原始健康数据、心理记录、学习记录或位置轨迹。

第六，可插拔行为插件层。  
运动、学习、睡眠、压力管理、职业技能、校园活动都可以作为上层插件接入。项目早期先选择最容易合规、最容易验证、最容易找到合作方的自律履约场景。

### 2.7 关于 AI 防作弊：有没有现成模型可以用

目前不存在一个可以直接购买、开箱即用、适配所有行为场景的“万能打卡防作弊模型”。原因很简单：运动、学习、睡眠、压力管理、线下活动的数据结构完全不同，作弊方式也不同。

但项目可以使用成熟的 AI 与机器学习组件，组合成早期可落地的风控系统：

| 能力 | 可用方法 | 早期实现方式 |
| --- | --- | --- |
| 时间序列异常检测 | Isolation Forest、One-Class SVM、AutoEncoder、统计阈值 | 检测短时突增、异常连续性、异常完成时间 |
| 图像/截图识别 | OCR、图片相似度、感知哈希、多模态模型 | 识别重复截图、伪造截图、模板化打卡图 |
| 文本打卡审核 | 大语言模型分类、关键词规则、语义相似度 | 识别复制粘贴、无意义文本、异常批量内容 |
| 设备数据校验 | 手机健康数据、穿戴设备数据、设备指纹、时间戳 | 提升运动、睡眠类凭证可信度 |
| 社区异常识别 | 群体行为分布、合作方历史通过率、异常通过率 | 给合作方建立见证质量评分 |
| 风险解释 | LLM 自动生成风控说明 | 让合作方知道被拦截原因，便于申诉 |

早期不需要追求“AI 全自动判真伪”。更务实的路线是：

1. 第一阶段：规则引擎 + 人工抽检 + 基础异常检测。
2. 第二阶段：积累样本后训练场景化风险评分模型。
3. 第三阶段：接入多模态模型和可信设备数据，形成 E2/E3/E4 分级凭证。

这也是面向政府评审更稳的表达：AI 用于提升效率和风控能力，而不是替代法律责任或制造不可解释的黑箱评价。

## 3. 目标用户与痛点

### 3.1 C 端用户：长期目标履约人群

核心人群：

- 大学生、研究生、高职学生；
- 初入职场人群；
- 备考、转行、技能训练人群；
- 运动健身、睡眠改善、自我管理用户；
- 有压力管理、情绪记录、心理韧性训练需求的用户。

他们的痛点：

- 长期坚持困难，缺少外部监督和即时反馈；
- 真实努力难以被看见，打卡记录沉没在各个平台中；
- 长期坚持无法沉淀成可展示、可复用的证明；
- 普通 App 只能记录数据，难以把真实履约转化为实际权益；
- 心理压力、作息混乱、运动不足等问题普遍存在，但用户不一定愿意进入严肃医疗场景。

项目提供的价值：

- AI 帮助拆解目标，降低行动门槛；
- 小组和社区见证提升坚持率；
- 行为凭证让用户的长期努力可以被授权展示；
- 用户可以获得品牌权益、课程优惠、活动名额、社群身份等实际回报；
- 非医疗化的压力管理与心理韧性训练，适合校园、训练营和自律履约场景。

### 3.2 B 端用户：品牌、机构、社区与校园组织

核心客户：

- 本地生活小微商户、个人经营者、社区门店、工作室；
- 运动品牌、健康食品、健身房、康复训练机构；
- 教育培训、职业技能、考试训练营；
- 高校社团、学生组织、创业园区、长期目标挑战活动；
- 心理韧性、压力管理、正念冥想、睡眠改善类社区；
- 企业工会、人力资源部门、员工健康管理活动。

他们的痛点：

- 传统投流成本高，平台广告存在大量无效曝光；
- 商户难以识别真正有行动力、有消费潜力、有持续需求的用户；
- 大量小微商户和个人经营者只需要做好本地小圈层生意，但缺少低成本识别、触达和激励潜在用户的工具；
- 训练营、社群和线下机构缺少标准化的用户行为证明；
- 校园活动和企业健康活动难以沉淀长期数据资产；
- 运营活动结束后用户关系容易流失，复购和转化困难。

项目提供的价值：

- 用行为凭证筛选更精准的人群，减少无效投放；
- 权益直接发给真实完成行为的用户，提升转化率；
- 帮助本地小微商户围绕地理位置、社区关系和用户行为特征做“小半径精准权益投放”；
- API 接入降低合作方开发成本；
- 数据看板帮助机构看到活动完成率、用户留存和权益领取效果；
- 标准化凭证可以跨活动、跨社区复用，形成长期用户资产。

### 3.3 政府与社会价值

项目符合数字经济、人工智能赋能实体经济、体育健康、校园创新创业、职业技能提升和数据要素合规流通等政策方向。

社会价值包括：

- 帮助大学生和年轻职场人建立健康生活方式、持续学习习惯和长期目标履约能力；
- 为校园社团、训练营、本地小微商户和个人经营者提供低成本数字化工具；
- 促进品牌预算从无效曝光转向真实用户权益；
- 支持本地生活服务、社区商业和个体经营者更高效地连接周边用户；
- 保护用户数据权益，推动用户授权、最小化使用和可验证凭证流通；
- 形成杭州本地可试点、可复制、可推广的可信行为凭证服务样板。

## 4. 市场分析

### 4.1 宏观基础

中国拥有庞大的移动互联网用户基础。根据 CNNIC 第 55 次《中国互联网络发展状况统计报告》，截至 2024 年 12 月，我国网民规模达到 11.08 亿人，互联网普及率为 78.6%，手机网民规模达到 11.05 亿人。移动互联网已经成为学习、运动、消费、社交和目标管理的基础设施。

高等教育人群规模足够支撑早期市场。教育部《2024 年全国教育事业发展统计公报》显示，全国各种形式的高等教育在学总规模为 4846.00 万人，全国高等学校 3119 所。高校人群目标明确、组织密度高、活动频繁、数字化接受度高，是项目最适合的冷启动人群。

AI 应用也正在从概念走向普及。生成式 AI 已经被用户用于问答、学习、办公和内容生成，政府和产业侧都鼓励 AI 与实体经济、教育、健康管理和公共服务结合。本项目将 AI 落在计划生成、风控审核和运营分析等具体环节，避免停留在概念包装。

### 4.2 C 端市场空间

项目早期不追求覆盖所有人，而是从三类高频场景切入：

- 运动自律：跑步、健身、减脂、体测、校园运动挑战；
- 学习履约：考研、考证、职业技能、阅读训练营；
- 心理韧性：压力管理、睡眠改善、正念练习、情绪记录。

以高校在学人群 4846 万人为基础，若早期只触达其中 1%，即约 48 万潜在用户；若其中 10% 愿意为 AI 计划、行为凭证和权益活动支付 19 元/月或购买低价年度会员，已经可以形成可观的种子收入。更重要的是，C 端用户的凭证会反向吸引 B 端商户进入权益投放网络。

早期 C 端付费策略应保持克制：

- 免费参与基础挑战；
- 高级 AI 计划、复盘报告、凭证皮肤、行为档案导出等收取低价会员；
- 不通过现金奖励刺激投机；
- 不设置质押、对赌、保本收益等金融化机制。

### 4.3 B 端市场空间

B 端是项目更重要的收入来源。原因是商户和机构本来就有营销、活动、用户运营和会员增长预算，只要项目能降低获客成本、提升转化，就有付费动力。

除品牌和机构客户外，项目还存在“本地微商业网络”的长期潜力。中国大量经营主体并不是大型品牌，而是社区门店、个体商户、工作室、健身教练、咖啡店、轻食店、瑜伽馆、培训老师、心理咨询工作室等小微经营者。他们通常没有能力长期在大平台购买高成本流量，也不需要全国性曝光，只需要在学校周边、社区周边、商圈周边找到一小批真实、有需求、可持续复购的用户。

Protocol: 75 可以为这类商户提供一种更轻量的获客方式：商户不再购买泛化曝光，而是基于用户授权展示的行为凭证，向特定小圈层用户发放本地权益。例如，学校周边轻食店可以面向“连续运动 30 天”的用户发放健康餐优惠；自习室可以面向“连续学习 21 天”的用户发放体验券；健身房可以面向“睡眠改善或运动自律”用户发放私教体验名额。这种模式不追求中心化大流量，而是服务本地半径内的多中心小生意。

可行收费方式：

| 收入类型 | 说明 | 早期价格参考 |
| --- | --- | --- |
| 活动 SaaS 服务费 | 为校园、社群、训练营创建挑战活动和凭证 | 3000-20000 元/场 |
| API 调用费 | 合作方调用凭证签发、验证、撤销接口 | 按调用量或套餐计费 |
| 品牌权益投放服务费 | 帮助品牌基于凭证筛选用户并发放权益 | 按活动或效果收服务费 |
| 本地商户权益工具 | 帮助小微商户创建本地优惠券、体验券和凭证验证活动 | 低价月费或按活动收费 |
| 数据看板服务费 | 向合作机构提供匿名化活动分析 | 按月或按项目收费 |
| 定制开发与运营服务 | 为高校、企业、健身房定制活动 | 按项目报价 |

早期可行的商业目标不是“全民信用基础设施”，而是在杭州及周边高校、创业园、健身房、训练营和目标管理社群中做出一批可展示案例，再逐步扩展到更多城市和行业。

## 5. 竞争分析

### 5.1 竞品与替代方案

项目不是没有竞品，但现有产品大多只解决其中一部分问题。

| 类型 | 代表产品/方案 | 优势 | 局限 |
| --- | --- | --- | --- |
| 运动健身 App | Keep、咕咚、薄荷健康等 | 用户基础大、内容成熟、设备数据较强 | 以内容和记录为主，跨平台凭证和 B 端权益验证较弱 |
| 学习打卡工具 | 番茄 ToDo、Forest、各类自习室 App | 高频、轻量、适合个人自律 | 行为真实性和商业凭证能力有限 |
| 社群/训练营工具 | 小打卡、知识星球、微信群打卡、小鹅通等 | 社区运营成熟，合作方易用 | 凭证标准不统一，难以跨社区验证 |
| 广告投放平台 | 抖音、小红书、朋友圈广告等 | 流量巨大，投放工具成熟 | 成本高、转化漏斗长，难以证明用户长期行为 |
| 信用类体系 | 芝麻信用等 | 社会认知强、数据规模大 | 属于大型平台级信用体系，不适合小团队复制，也不适合本项目做金融化延伸 |
| 区块链存证平台 | 联盟链、电子数据存证服务等 | 存证可信、技术成熟 | 只能证明数据未被篡改，无法单独证明行为真实 |

### 5.2 我们的差异化

第一，不做单一 App，而做凭证基础设施。  
上层行为场景可以变化，底层凭证生成、验证、存证和授权展示能力可以复用。

第二，不做大而全信用分，而做有限事实证明。  
平台不评价一个人“好不好”，只证明某人在某规则下完成过某项行为，降低合规风险并提升可信度。

第三，不把区块链作为全部卖点。  
区块链或外部存证只是 A3 等级的增强选项，项目核心是证据分层、AI 风控、多方签名和用户授权。

第四，既服务 C 端长期目标履约，也服务 B 端降本增效。  
用户获得行为证明和权益，商户减少无效投放，平台获得 SaaS/API/运营服务收入。

第五，适合小团队冷启动。  
项目可以先接入已有社群和机构，不必一开始烧钱做全民 App。合作方提供用户和场景，平台提供可信凭证能力。

### 5.3 核心壁垒

- 凭证等级体系：E/A 二维分层让不同价值权益匹配不同可信度，降低作恶收益。
- 合作方网络：越多社群、校园和机构接入，凭证越有流通价值。
- 风控数据积累：行为样本越多，AI 异常识别和合作方评级越准确。
- 隐私合规能力：用户授权、最小化披露、可撤销凭证会成为长期信任基础。
- 场景标准化能力：把不同活动抽象为统一凭证协议，形成可复制的行业模板。

## 6. 风险控制与合规设计

### 6.1 不做个人征信

项目名称和对外话术中避免使用“个人信用评分”“征信”“授信”等表述。平台只提供“可信行为凭证”，不向金融机构输出用于贷款、授信、保险定价或其他金融风控的个人信用评价。

凭证表达的是有限事实：

> 某用户在某时间段、某规则下、基于某类证据完成过某项行为。

这与传统征信业务中的个人信用信息采集、加工、评价和提供有明确区别。

### 6.2 不做金融业务

项目不涉及：

- 代币发行；
- 稳定币；
- 虚拟货币交易；
- 用户质押；
- 对赌罚金；
- 底层生息；
- 保本收益；
- 资金池；
- 金融投资回报承诺。

用户获得的是优惠券、服务权益、活动资格、会员体验、品牌礼包等非金融权益。平台收入来自技术服务费、SaaS 服务费、API 调用费和运营服务费。

### 6.3 资金转移风险控制

早期不将商户权益预算沉淀在公司账户中，避免形成代收代付或类资金池风险。

推荐三种模式：

| 模式 | 资金路径 | 合规优势 |
| --- | --- | --- |
| 权益券模式 | 商户发放优惠券、体验券、课程名额、活动资格 | 平台不触碰现金，最适合早期 |
| 商户直付模式 | 平台验证凭证后，商户通过自己的支付或会员系统发放 | 平台只提供技术结果，不转移资金 |
| 持牌机构托管模式 | 高价值现金激励由银行或持牌支付机构托管 | 平台只提交释放条件，不能挪用资金 |

项目早期坚决不做“商户把钱打给平台，平台再分给用户”的模式。

### 6.4 个人信息与敏感数据保护

运动、健康、睡眠、心理状态、位置轨迹等数据可能涉及敏感个人信息，必须采用更高标准保护。

合规措施：

- 明示目的、范围、保存期限和使用方式；
- 对敏感个人信息取得单独同意；
- 坚持最小必要原则，只采集生成凭证所需数据；
- 商户默认只看到验证结果，不看到原始数据；
- 用户可以撤回授权、删除非必要数据、申请凭证撤销；
- 未成年人场景早期不主动开展，后续如进入校园未成年人场景需引入监护人同意和更严格保护；
- 心理韧性场景不做诊断、治疗和医疗建议，只做压力管理、情绪记录和自助练习；如识别高风险表达，应提示寻求专业机构帮助。

### 6.5 与传统区块链项目的区别

本项目不依赖公链、不发行代币、不做链上金融，不把“上链”作为唯一可信来源。

项目的可信来自四个层次：

1. 证据来源：用户、社区、设备、机构等多方数据。
2. AI 风控：异常检测、重复识别、风险评分。
3. 多方签名：用户授权、合作方见证、平台校验分别签名。
4. 存证机制：哈希日志、Merkle Root、第三方时间戳或合规联盟链锚定。

区块链只在高等级存证中作为 A3 选项之一，用来证明凭证包在某个时间点已经存在且未被篡改，而不是证明行为本身天然真实。

如未来以区块链技术或系统向社会公众提供信息服务，项目将按《区块链信息服务管理规定》履行备案、实名、安全管理、日志留存和内容治理义务。

### 6.6 技术风险控制

AI 风控不承诺百分之百识别作弊。平台通过等级体系和申诉机制控制风险：

- 低证据凭证只能兑换低价值权益；
- 高价值权益要求 E3/E4 与 A2/A3；
- 合作方有历史质量评分，异常通过率高的合作方降级或暂停；
- 高风险凭证可进入人工复核；
- 凭证支持撤销和过期；
- 商户可配置接受等级和风控阈值。

### 6.7 成本与可行性控制

三人团队早期可完成 MVP，原因是项目采用轻资产路线：

- 前端优先使用微信小程序和 Web 后台；
- 后端使用成熟云服务、数据库、队列、对象存储；
- AI 早期使用大模型 API + 开源机器学习算法 + 规则引擎；
- 存证早期先做 A1/A2，不急于接入高成本链上服务；
- 用户增长先借助合作社群和校园活动，不依赖大规模投放。

项目早期核心投入集中在产品开发、试点运营、服务器、安全合规和软著申请上，5 万元启动资金可以覆盖 MVP 与第一批试点成本。

## 7. 商业模式

### 7.1 收入结构

| 收入来源 | 客户 | 说明 |
| --- | --- | --- |
| SaaS 活动服务费 | 高校社团、训练营、健身房、企业 | 创建活动、管理报名、生成凭证、查看看板 |
| API 调用费 | 社区平台、课程机构、应用开发者 | 签发、验证、撤销、查询凭证 |
| 品牌权益投放服务费 | 运动、健康、教育、生活方式品牌 | 基于凭证筛选用户并发放权益 |
| 本地商户权益工具 | 社区门店、个体商户、工作室 | 创建本地优惠券、体验券、核销码和凭证验证活动 |
| C 端会员 | 长期目标履约用户 | AI 计划、复盘报告、凭证档案、进阶挑战 |
| 定制运营服务 | 企业、高校、园区 | 定制挑战方案、数据报告、现场活动 |

### 7.2 早期收费策略

早期不追求高客单价，而追求案例可验证：

- 校园活动：低价或免费试点，换取真实案例和用户反馈；
- 社群/训练营：按活动收费 3000-10000 元；
- 品牌活动：按权益投放项目收费 5000-20000 元；
- API：先对合作方免费或低价开放，积累调用量；
- C 端：先免费，后续推出 9.9-29.9 元/月的 AI 履约会员。

### 7.3 为什么商户愿意付费

传统投流的核心问题是“先买曝光，再赌转化”。本项目的逻辑是“先验证行为，再发放权益”。

商户愿意付费的理由：

- 用户已经完成某类行为，需求更明确；
- 权益发给真实完成者，减少羊毛党和无效曝光；
- 凭证等级可配置，预算可控；
- 平台提供活动数据看板，方便复盘 ROI；
- 商户不需要自己开发复杂的打卡、风控和凭证系统。
- 对本地小微商户而言，可以把投放范围控制在自己能服务的地理半径内，用更小预算触达更可能到店和复购的人。

### 7.4 本地微商业网络的长期潜力

Protocol: 75 的去中心化表达，不是让每个商户都接触复杂的链上系统，而是让权益投放从单一大平台流量入口，转向由许多社区、学校、门店、工作室和训练营共同组成的多中心网络。

在这个网络里，用户凭证由用户授权展示，商户可以自主选择接受哪些凭证等级，平台只提供验证、风控、核销和数据看板能力。这样既保留了“用户拥有自己的行为证明”的思想，也让本地小微商户拥有一种更公平的数字化获客工具。

早期该方向不作为主要收入假设，而作为项目的中长期商业潜力：当凭证数量、合作方数量和本地权益供给增加后，Protocol: 75 可以逐步形成围绕学校、社区、商圈和兴趣社群的小范围权益网络。

目标：做出可演示、可试点、可答辩的最小可行产品。

- 建立 B 端商户权益投放后台；

- 与合规存证服务或联盟链服务对接，形成 A3 试点；
- 建立隐私合规、数据安全、凭证撤销和争议处理流程。

- 3-5 个可复制行业模板；

- 月度经常性收入开始形成。

目标：形成长期目标履约场景下的可信行为凭证网络。

## 11. 政府补贴答辩口径

### 11.1 推荐表达

我们做的是“AI + 可信持续行为凭证 + 长期目标履约服务”。项目帮助用户把长期坚持转化为可验证的行为证明，帮助商户和机构减少无效投放，把预算用于真实完成行为的用户权益。项目早期聚焦高校、社群和训练营，技术上采用 AI 风控、用户授权、多方签名和可信存证，商业上采用 SaaS、API 和权益投放服务费，合规上不做金融、不做征信、不碰资金池。

### 11.2 不推荐表达

答辩时不建议使用以下表达：

- Web3 金融协议；
- 链上信用；
- 稳定币、质押、对赌、罚金、底层生息；
- 个人征信、信用评分、金融信用；
- 用户投资回报；
- 平台托管商户资金；
- 心理疾病治疗、诊断、康复承诺。

### 11.3 评委可能问的问题

问题：你们和普通打卡 App 有什么不同？  
回答：普通打卡 App 主要记录用户行为，我们重点做可验证凭证。用户完成的长期行为可以在授权后被商户和机构验证，形成权益领取和商业合作闭环。

问题：你们是不是在做征信？  
回答：不是。我们不评价个人信用，不服务金融授信，不输出金融信用分。我们只证明有限事实，例如某用户在某规则下完成过 30 天运动挑战。

问题：你们是否涉及金融或资金池？  
回答：不涉及。我们不做代币、质押、对赌、收益或资金池。商户权益主要是优惠券、服务名额、会员体验等；如涉及现金奖励，也由商户直付或持牌机构托管。

问题：区块链在项目里是什么作用？  
回答：区块链不是项目唯一底层，也不是金融工具。它只是高等级凭证的外部存证选项，用于证明记录没有被事后篡改。项目核心是 AI 风控、证据分层、多方签名和用户授权。

问题：AI 如何防作弊？  
回答：我们不依赖单一模型，而是采用规则引擎、异常检测、图片识别、文本审核、设备数据校验和人工抽检组合。不同证据质量对应不同凭证等级，高价值权益需要更高等级凭证。

问题：三人团队如何启动？  
回答：我们早期不做重资产 App，而是做小程序、合作方后台和凭证 API，接入已有社群和校园活动。第一阶段 3 个月内可完成 MVP，5 万元资金足够支撑产品原型、试点运营和软著申请。

## 12. 结论

Protocol: 75 的本质不是“自律 App”，而是“可信持续行为凭证协议”。项目保留了去中心化思想中最有价值的一部分：让用户拥有自己的行为证明，让小团队也能提供可验证、可审计、可授权流通的行为凭证。

在中国大陆落地时，项目主动剥离金融化、代币化和投机化表达，转向 AI 赋能长期目标履约、数据合规流通、商户降本增效和实体权益直达。该方向具备技术创新性、商业可行性、社会价值和合规可控性，适合作为大学生创业补贴项目进行申报和答辩。
